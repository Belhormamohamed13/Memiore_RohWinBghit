\chapter*{Résumé}
\addcontentsline{toc}{chapter}{Résumé}
\thispagestyle{plain}

Ce mémoire présente la conception et la réalisation de \textit{RohWinBghit}, une plateforme mobile de covoiturage inter-wilayas dédiée au marché algérien. Il répond à une problématique réelle~: l'absence d'une solution numérique sécurisée pour les déplacements longue distance entre les 69~wilayas, marqués par un coût élevé, une offre limitée et un déficit de confiance entre usagers.

L'architecture proposée combine une application mobile \textit{React Native}~0.81 (Expo SDK~54), un \textit{back-end} Express.js fondé sur Knex.js et PostgreSQL, un cache Redis, et un service IA \textit{FastAPI} dédié à la vérification biométrique d'identité (KYC) intégrant détection de vivacité et module anti-\textit{spoofing}. L'application offre une authentification par OTP, un système de réservation temps réel, un suivi GPS via Mapbox, une messagerie instantanée Socket.IO, un mécanisme expérimental de tarification dynamique, un module de détection de fraude et un moteur de paiement multi-méthodes implémenté via un patron \textit{Strategy/Factory}~: le règlement en espèces est opérationnel, tandis que CIB et Edahabia sont intégrés en environnement \textit{sandbox} dans l'attente de l'homologation SATIM.

La validation expérimentale repose sur une suite automatisée \textit{Jest} comportant 107~tests unitaires (authentification, tarification, score conducteur, détection de fraude, machine à états des trajets) réussis à 100\,\%, complétés par 100~tests d'intégration HTTP/Socket.IO et par une campagne de validation manuelle portant sur 62~écrans distincts.

\vspace{0.3cm}
\textbf{Mots-clés~:} covoiturage, React Native, Express.js, vérification biométrique KYC, PostgreSQL, Redis, paiement mobile, Algérie, inter-wilayas.

\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
\thispagestyle{plain}

This thesis presents the design and implementation of \textit{RohWinBghit}, a mobile carpooling platform tailored to the Algerian inter-wilaya transportation market. It addresses a real societal issue~: the absence of a secure, trusted digital solution for long-distance travel between Algeria's 69~wilayas, characterised by high fares, inconsistent availability and a pervasive lack of user trust.

The proposed architecture combines a \textit{React Native}~0.81 mobile application (Expo SDK~54), an Express.js back-end built on Knex.js and PostgreSQL, a Redis cache layer, and a dedicated \textit{FastAPI} AI service for biometric identity verification (KYC) featuring liveness detection and an anti-\textit{spoofing} module. The application provides OTP authentication, real-time booking, Mapbox-based GPS tracking, Socket.IO instant messaging, an experimental dynamic pricing mechanism, a fraud-detection module and a multi-method payment engine implemented through a \textit{Strategy/Factory} pattern: cash settlement is fully operational, while CIB and Edahabia are integrated in \textit{sandbox} mode pending SATIM clearance.

Experimental validation relies on an automated \textit{Jest} suite: 107~unit tests covering the core business logic (authentication, pricing, driver scoring, fraud detection, trip state machine) all pass with a 100\,\% success rate, complemented by 100~HTTP/Socket.IO integration tests and a manual validation campaign covering 62~distinct application screens.

\vspace{0.3cm}
\textbf{Keywords:} carpooling, React Native, Express.js, Knex.js, biometric verification, PostgreSQL, Redis, artificial intelligence, mobile payment, Algeria, inter-wilaya.
